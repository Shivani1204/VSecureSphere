Theory

The purpose of this experiment is to understand how to use Nmap (Network Mapper) to scan a network and gather information about active devices, open ports, and services running on those devices.

What is Nmap?
Nmap is a free, open-source tool designed for network discovery and security auditing. It is widely used by network administrators, security professionals, and ethical hackers to analyze network security and identify vulnerabilities. Some key functions of Nmap include:

  • Discovering hosts (devices) on a network.
  • Detecting open ports.
  • Identifying running services and their versions.
  • Performing OS detection.
  • Detecting firewalls and security mechanisms.

Understanding Ports and Services
In networking, a port is a logical communication endpoint that allows devices to send and receive data. Think of ports as doors in a building—each door (port) leads to a different service or function.

Port Numbers and Their Uses:

  • Port 22 (SSH): Secure remote login.
  • Port 80 (HTTP): Unencrypted web traffic.
  • Port 443 (HTTPS): Secure web browsing.
  • Port 25 (SMTP): Email transfer.
  • Port 53 (DNS): Domain name resolution.

Types of Port States in Nmap
When scanning a target, Nmap categorizes ports into different states:

  • Open: The port is accessible and a service is actively listening.
  • Closed: The port is reachable but no service is running.
  • Filtered: A firewall or security mechanism is blocking access.
  • Unfiltered: The port is accessible, but Nmap can't determine its state.

How Nmap Works
Nmap sends specially crafted packets to the target system and analyzes the responses to determine:

  • The list of live hosts on a network.
  • The operating system running on a device.
  • The services and their versions running on open ports.
  • Firewall and security settings.

Common Nmap Scan Types

  • Ping Scan (-sn): Detects active hosts without scanning ports.
  • SYN Scan (-sS): Stealth scan that doesn’t complete the handshake, making it harder to detect.
  • TCP Connect Scan (-sT): Fully establishes a connection before scanning.
  • UDP Scan (-sU): Detects open UDP ports (used for DNS, SNMP, etc.).
  • Service Version Detection (-sV): Identifies versions of running services.
  • Operating System Detection (-O): Attempts to determine the OS of the target.

By using Nmap effectively, we can analyze network security, identify potential vulnerabilities, and ensure that only necessary services are exposed to the internet.
