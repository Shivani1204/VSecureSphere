Welcome to Experiment 8: Configuring a Firewall using IPTables

Introduction to Firewalls and IPTables

A firewall is a security mechanism that controls incoming and outgoing network traffic based on predetermined security rules. It serves as a barrier between a trusted internal network and untrusted external networks, such as the internet. Firewalls can be hardware-based, software-based, or a combination of both and are essential in safeguarding systems from unauthorized access and cyber threats.

One of the most commonly used software-based firewalls on Linux systems is IPTables, a tool that enables administrators to define firewall rules for packet filtering and network security.

Understanding IPTables

IPTables is a packet filtering framework built into the Linux kernel that enables the creation of rules to control, filter, and manipulate network traffic. It works on the principle of tables, chains, and rules to determine how packets should be handled.

Key Tables in IPTables:

  • Filter Table – Responsible for filtering packets (Default Table).
  • NAT Table – Handles network address translation (NAT) for modifying source/destination IP addresses.
  • Mangle Table – Used for specialized packet alterations (e.g., modifying headers for Quality of Service).

Chains in the Filter Table:

  • INPUT – Manages packets destined for the local machine.
  • FORWARD – Handles packets routed through the machine.
  • OUTPUT – Controls packets sent from the machine.

Common Use Cases of IPTables:

  • Blocking unauthorized access by setting default policies.
  • Allowing or restricting specific services, such as SSH and HTTP.
  • Preventing DoS attacks by limiting connections.
  • Restricting ping requests to prevent network enumeration.

IPTables allows network administrators to enforce security policies and define granular access controls, making it a powerful firewall tool for system and network security.
