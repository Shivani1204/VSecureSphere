Procedure

Step 1: Install hping

To install hping, update your package list and install hping using the following command:

            sudo apt-get update
            sudo apt-get install hping3

Type the above command in the terminal.

Step 2: Check if a Host is Alive (ICMP Ping)

Use hping to send an ICMP echo request (similar to the ping command):

            hping3 -1

This command sends an ICMP echo request to check if the target is online.

Step 3: Perform a TCP SYN Scan

To check for open ports on a target system, use the SYN scan:

            hping3 -S -p

The -S flag sends a SYN packet, and -p specifies the target port.

Step 4: Perform a TCP Connect Scan

To fully establish a connection with a target system, use the following command:

            hping3 -S -p  --tcp-connect

This method completes the full TCP handshake, unlike a SYN scan.

Step 5: Scan for Open UDP Ports

To scan UDP ports (e.g., DNS on port 53), use:

            hping3 --udp -p 53

This command sends UDP packets to the target.

Step 6: Conduct a Flood Attack Simulation

To test how a target system handles a high rate of requests:

            hping3 -S --flood -p

Warning: This generates a large number of packets and should only be used in a controlled lab environment.

Step 7: Example Output

After executing these commands, you may see output like:

            HPING  (eth0 ): S set, 40 headers + 0 data bytes
            len=46 ip= ttl=64 DF id=0 sport=80 flags=SA seq=0 win=65535

Explanation:

  • SA (SYN-ACK): Indicates that the port is open.
  • RA (RST-ACK): Means the port is closed.
  • Flags: Shows response types for TCP analysis.
