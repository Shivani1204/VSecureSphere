Wireshark Network Traffic Analysis Experiment

In this experiment, you will use Wireshark, a network protocol analyzer, to observe, capture, and analyze network traffic. Follow each step carefully and note your observations.

1. Login to the System

  1. Open the noVNC interface using the Minikube service URL provided by your instructor.
  2. Enter the password below when prompted:
    3. Password: VSecureSphere

2. Open a Terminal Window

  1. Locate the Terminal application on the desktop or in the taskbar.
  2. Click on it to open a command-line interface (CLI), where you’ll be able to start Wireshark and execute other network commands.

3. Launch Wireshark

wireshark &

Wireshark should open in a new window. This tool will allow you to capture and analyze packets traveling through the network interface.

4. Select a Network Interface for Packet Capture

  1. When Wireshark opens, you’ll see a list of available network interfaces (such as eth0, lo, etc.).
  2. Select the main network interface (usually eth0 or lo if testing locally) by double-clicking on it to start capturing live network traffic.

5. Generate Network Traffic

Now that packet capture is running, let’s generate some network traffic for analysis. Perform the following actions in a new terminal window:

  1. Ping a Website: Use the ping command to send ICMP packets to a website:
    2. ping google.com -c 5

    This will send five ICMP packets to google.com and display responses in the terminal. Observe the packets appearing in the Wireshark capture window.

  2. Fetch a Web Page: To generate HTTP traffic, use the curl command:
    3. curl http://example.com

    This command sends an HTTP request to example.com and returns the HTML content of the page in the terminal. Look for HTTP packets in Wireshark.

  3. DNS Query: Perform a DNS lookup to resolve a domain name:
    4. nslookup google.com

    This command queries a DNS server to resolve the IP address of google.com. You should see DNS packets in Wireshark.

6. Analyzing the Captured Packets

  1. Return to the Wireshark window, where you’ll see packets appearing in real time.
  2. Stop the Capture by clicking the red square button in the Wireshark toolbar once you have a few packets.
  3. The Wireshark window has three main sections:
    • Packet List Pane (top): Displays a list of all captured packets with basic details (time, source, destination, protocol).
    • Packet Details Pane (middle): Shows detailed information about the selected packet, organized by protocol layers (Ethernet, IP, TCP/UDP, etc.).
    • Packet Bytes Pane (bottom): Displays the raw data of the selected packet in hexadecimal and ASCII formats.
  4. Explore Each Section:
    • Click on different packets in the Packet List to view their details.
    • In the Packet Details pane, expand each protocol layer (such as Ethernet, IP, TCP) to view more information.
    • The Packet Bytes pane shows the raw binary data, which is often useful for deeper analysis.

7. Applying Filters for Specific Traffic

Wireshark allows you to apply filters to focus on specific types of network traffic. Let’s try a few:

  1. Filter by Protocol:
    • ICMP Traffic (Ping): Type icmp in the Display Filter box at the top and press Enter.
    • HTTP Traffic: Type http and press Enter.
    • DNS Traffic: Type dns and press Enter.
  2. Filter by IP Address: Use the following format to view only packets coming from or going to a specific IP address:
    3. ip.addr == <IP_ADDRESS>

    Replace <IP_ADDRESS> with the actual IP (e.g., ip.addr == 8.8.8.8).

  3. Experiment with Filters: Apply each filter one by one and observe how the packet list changes to match the specified criteria.

8. Save Your Packet Capture

  1. Once you have completed the capture and analysis, save your work for further review.
  2. Go to File > Save As in the Wireshark menu.
  3. Choose a filename and location to save the capture in .pcapng format.

9. Close Wireshark

  1. To exit Wireshark, select File > Quit or click the close button on the window.
  2. Make sure you have saved your capture file if you want to review it later.
