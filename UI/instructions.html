Nmap Experiment Instructions

Step 1: Login to Both Systems

  • Attacker System: Password: VSecureSphere
  • Victim System: Password: VSecureSphere

Step 2: Configure Network Connectivity

Check the IP addresses of both the attacker and victim systems using the following command:

ifconfig

Note the IP address of the victim system for further steps.

Step 3: Verify Connectivity Between Attacker and Victim

From the attacker system, ping the victim system to confirm they are on the same network:

ping <victim_IP>

Replace <victim_IP> with the actual IP address of the victim system.

Step 4: Perform Basic Nmap Scans

  • Basic Ping Scan: Checks if the victim system is online. 
    nmap -sn <victim_IP>
  • Simple Port Scan: Lists open ports and services on the victim system. 
    nmap <victim_IP>
  • Service Version Detection: Identifies the version of services running. 
    nmap -sV <victim_IP>
  • Operating System Detection: Detects the victim's OS. 
    nmap -O <victim_IP>
  • Aggressive Scan: Comprehensive scan to gather detailed information. 
    nmap -A <victim_IP>

Step 5: Scan Specific Ports

If you want to scan specific ports (e.g., ports 80 and 443), use:

nmap -p 80,443 <victim_IP>

To scan a range of ports (e.g., 1 to 1000):

nmap -p 1-1000 <victim_IP>

Step 6: Detect Vulnerabilities

To detect known vulnerabilities on the victim system, use the following command:

nmap --script vuln <victim_IP>

Step 7: Advanced Scans

  • TCP SYN Scan: Detects open ports. 
    nmap -sS <victim_IP>
  • UDP Scan: Scans for open UDP ports. 
    sudo nmap -sU <victim_IP>
  • Scan All Ports: Scans all 65,535 ports. 
    nmap -p- <victim_IP>
  • Stealthy Scan: Attempts to evade detection. 
    nmap -sS <victim_IP>

Step 8: Save Scan Results

You can save your Nmap scan results in a file for later analysis:

nmap -oN scan_results.txt <victim_IP>

To save the results in XML format:

nmap -oX scan_results.xml <victim_IP>
