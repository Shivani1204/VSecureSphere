Procedure

Step 1: Launch SEToolkit

SEToolkit is a powerful framework for simulating real-world social engineering attacks. To launch it, open the terminal and enter:

            sudo setoolkit

You may be prompted to agree to the terms and conditions â€” type yes to continue.

Step 2: Select Social Engineering Attacks

From the SET main menu, choose option:

            1) Social-Engineering Attacks

Step 3: Select Website Attack Vectors

Next, choose the method to clone a website for the phishing simulation:

            2) Website Attack Vectors

Step 4: Select Credential Harvester Attack Method

This method allows us to clone a website and capture credentials:

            3) Credential Harvester Attack Method

Step 5: Select Site Cloner

Choose the Site Cloner option to clone a legitimate website (like Facebook, Gmail, etc.):

            2) Site Cloner

Step 6: Enter the Attack Parameters

When prompted:

  • Enter your **local IP address** (you can find it using ifconfig).
  • Enter the **URL of the site** you want to clone (e.g., https://accounts.google.com).

SEToolkit will clone the login page and set up a listener on your IP to capture credentials.

Step 7: Phishing Page Deployment

The fake login page will be hosted locally (usually on port 80). Share your IP with the victim (only in a controlled lab setup), and once they enter details, the credentials will be stored locally.

Step 8: View Captured Credentials

Captured usernames and passwords are stored in a file at:

            /var/www/html/harvester_*.txt

Use cat or nano to view the captured credentials.

Step 9: Conclude the Experiment

Discuss how phishing attacks work in real-world scenarios and how users can avoid falling victim to such techniques.

  • Always check the URL and SSL certificate.
  • Never click on suspicious links from emails or messages.
  • Enable two-factor authentication for better security.
