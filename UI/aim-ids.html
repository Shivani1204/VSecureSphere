Aim

To simulate a network-based intrusion using tools like hping3 and analyze detection using Snort IDS on the victim system.


Prerequisites

  • Basic understanding of Intrusion Detection Systems (IDS), especially Snort.
  • Familiarity with Linux command-line operations.
  • Knowledge of common penetration testing tools such as hping3 and Nmap.
  • Both Attacker (Kali) and Victim (Ubuntu with Snort) containers must be running and accessible via browser-based noVNC terminals.
  • Docker, Kubernetes (or Minikube), and noVNC should be properly set up and configured.
  • The attacker container should have necessary capabilities (e.g., --privileged) for generating raw packets.
  • The victim container should be running Snort configured with appropriate rules and listening on the correct network interface.

Tools Required

  • hping3 - A network tool used for crafting custom packets and simulating network-based attacks.
  • Snort IDS - An open-source intrusion detection and prevention system used to monitor network traffic for suspicious activity.
  • Docker - A platform used for running the attacker and victim containers in isolated environments.
  • noVNC - A tool used to access remote Linux desktops via a browser, allowing interaction with containers.
  • Kali Linux - The attacker machine, equipped with tools for penetration testing and security analysis.
  • Ubuntu - The victim machine, running Snort to detect and analyze intrusions.

Learning Outcomes

  • Understand the basic concepts of Intrusion Detection Systems and their role in network security.
  • Learn how to simulate network-based intrusions using tools like hping3.
  • Gain practical experience in configuring Snort IDS for network traffic analysis.
  • Analyze the effectiveness of IDS in detecting simulated intrusions.
  • Understand how to fine-tune IDS rules to improve detection accuracy.
  • Develop skills in using containers and noVNC for conducting penetration testing in a virtual environment.

Safety Precautions

  • Ensure the experiment is performed in a controlled lab environment or isolated test network.
  • Only use authorized systems for conducting intrusion simulations.
  • Monitor traffic carefully to avoid unintentional network disruptions.
  • Ensure that the Snort IDS rules are properly configured to avoid unnecessary alerts.
  • Follow ethical guidelines when simulating intrusions and ensure that the tests are non-destructive.
  • Document the process, settings, and findings for educational purposes.
