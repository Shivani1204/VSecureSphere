Procedure

Step 1: Install John the Ripper

John the Ripper is a powerful password cracking tool. To install it, open the terminal and enter the following command:

            sudo apt-get install john

After installation, verify that John the Ripper is successfully installed by checking its version:

            john --version

Step 2: Prepare the Password Hashes

Before you can start cracking passwords, you need to have password hashes. These can be extracted from various systems like Linux, Windows, or web applications. For this experiment, we will use a sample password hash. If you are using Linux, you can extract password hashes from the /etc/shadow file.

Alternatively, use a hash generated with a tool like openssl:

            echo -n "password123" | openssl dgst -sha256

This will produce a SHA-256 hash of the string "password123".

Step 3: Prepare a Wordlist

John the Ripper uses wordlists (lists of possible passwords) to perform dictionary attacks. You can use a pre-built wordlist or create your own. For this example, we will use the popular rockyou.txt wordlist.

Download the rockyou.txt wordlist if it's not already available on your system:

            sudo apt-get install wordlists

The wordlist can be found in the /usr/share/wordlists/rockyou.txt.gz directory. Unzip it if needed:

            gzip -d /usr/share/wordlists/rockyou.txt.gz

Step 4: Run John the Ripper

Once you have the password hash and wordlist ready, run John the Ripper on the hash file. Create a file containing the hash (e.g., password_hash.txt) and use the following command to start cracking:

            john --wordlist=/usr/share/wordlists/rockyou.txt password_hash.txt

John will begin checking the hashes against the wordlist to find a match.

Step 5: Monitor the Cracking Process

John will display the cracking process, showing how long each attempt took and how many passwords it has checked. You can view the progress of John the Ripper in real-time.

Step 6: View the Cracked Password

Once the password is found, John will display it on the screen. You can also view the cracked password by running:

            john --show password_hash.txt

Step 7: Conclude the Experiment

After cracking the password, discuss the following:

  • How password strength affects cracking time.
  • The importance of using complex, unpredictable passwords to prevent brute-force and dictionary attacks.
  • Ways to defend against password cracking, such as using multi-factor authentication (MFA) and secure hashing algorithms.

Always remember that password cracking should only be performed in controlled, ethical environments, such as penetration testing and security research.
