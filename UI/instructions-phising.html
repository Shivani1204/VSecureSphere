Phishing Experiment Using SET and Docker

This experiment demonstrates how phishing attacks are simulated using Social Engineering Toolkit (SET) inside a Docker container.

Step 1: Login to Both Systems

  • Attacker System: Password: VSecureSphere
  • Victim System: Password: VSecureSphere

Step 2: Start Social Engineering Toolkit (SET)

Update the package lists to get the latest versions of packages and dependencies:

apt-get update

Install the python3-pexpect package, which allows SET to interact with terminal-based programs:

apt-get install python3-pexpect -y

Navigate to the SEToolkit directory:

cd /opt/setoolkit

Install the Social Engineering Toolkit using the Python setup script:

sudo python3 setup.py install

Start the SET interface:

sudo setoolkit

After running the above commands, you will see the SET main menu. Follow the steps below to perform a phishing attack.

Step 3: Choose Social Engineering Attacks

In the SET main menu, choose the Social-Engineering Attacks option: 1

Step 4: Select Website Attack Vectors

Select the Website Attack Vectors option: 2

Step 5: Select Credential Harvester Attack Method

Select the Credential Harvester Attack Method: 3

Step 6: Choose Site Cloner

Select the Site Cloner option to mirror a real website: 2

Step 7: Enter Attacker IP

When prompted for "What is the IP address to use for the POST back?", enter the attacker's container IP. You can find it using:

ifconfig

Step 8: Enter the Website URL to Clone

Enter a real login page to clone, such as:

https://accounts.google.com

SET will clone the site and host it on port 80.

Step 9: Access the Fake Site from the Victim

On the victim's machine, open a browser and enter the attacker's IP to access the fake login page, e.g.:

http://192.168.49.1

Step 10: Capture Credentials

Once the victim enters their credentials, SET will capture and store them. You can find the results in:

/root/.set/reports/<date-folder>/
