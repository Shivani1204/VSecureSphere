Procedure

Step 1: Install Nessus

Before you can start scanning, you need to install Nessus. On Kali Linux, use the following command to install Nessus:

            sudo apt install nessus

Once installed, start the Nessus service:

            sudo systemctl start nessusd

You can check the status of Nessus with:

            sudo systemctl status nessusd

Once Nessus is running, open a browser and go to https://localhost:8834 to access the Nessus web interface.

Step 2: Set Up an Account

When you access Nessus for the first time, you'll be prompted to set up an account. Provide your email and create a password to log in.

Step 3: Start a New Scan

Once logged in, you’ll be on the Nessus dashboard. To create a new scan, click on the New Scan button.

Choose the type of scan you want to run. For a basic vulnerability scan, select Basic Network Scan.

Step 4: Configure Scan Settings

In the scan configuration page:

  • Enter the target IP address or range of the system(s) you want to scan.
  • Optionally, configure authentication or credentialed scans if needed.
  • Set any advanced scan options as required (e.g., scan ports, detection settings).

Step 5: Start the Scan

Once the scan is configured, click Save and then Launch to start the scan. Nessus will begin scanning the target system for vulnerabilities.

Step 6: Monitor Scan Progress

You can monitor the progress of the scan in real-time from the Nessus dashboard. The scan may take a few minutes depending on the number of systems being scanned and the complexity of the scan.

Step 7: Review Scan Results

Once the scan is complete, you’ll be able to see the results in the scan report. The results will include:

  • Vulnerabilities: A list of discovered vulnerabilities, categorized by severity (critical, high, medium, low).
  • Detailed Description: Information about each vulnerability, including its impact and possible exploits.
  • Remediation Advice: Recommendations on how to fix or mitigate each vulnerability.

Step 8: Address the Vulnerabilities

Based on the scan results, prioritize and address the vulnerabilities with the highest severity. Follow the remediation advice to patch or fix the vulnerabilities. Some common steps include:

  • Applying security patches to the system.
  • Disabling unnecessary services or ports.
  • Changing weak or default passwords.

Step 9: Re-scan After Remediation

After remediating the vulnerabilities, re-scan the system to ensure that the issues have been resolved. This will help confirm the effectiveness of your fixes.

Step 10: Conclude the Experiment

Once all vulnerabilities have been addressed and the system is secure, conclude the experiment by discussing:

  • The importance of regular vulnerability scanning in maintaining system security.
  • The role of Nessus in identifying security risks and helping with remediation.
  • How vulnerability management contributes to an overall cybersecurity strategy.
