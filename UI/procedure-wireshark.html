Procedure

Step 1: Install Wireshark

To install Wireshark on a Linux system, open the terminal and execute the following commands: 

            sudo apt-get update
            sudo apt-get install wireshark

This updates the package list and installs Wireshark. During installation, you may be prompted to allow non-root users to capture packets.

Step 2: Launch Wireshark

Start Wireshark with the necessary permissions using the command: 

            sudo wireshark

Running Wireshark with `sudo` ensures that it has the required permissions to capture network traffic.

Step 3: Select a Network Interface

Once Wireshark opens, you will see a list of available network interfaces. Choose the appropriate interface (e.g., Wi-Fi, Ethernet) to monitor traffic.

Step 4: Start Capturing Packets

Click the "Start" button to begin capturing network traffic. Wireshark will now display live packet data in real time.

Step 5: Analyze Captured Packets

Once sufficient traffic has been captured, click the "Stop" button to halt the capture. Wireshark will display captured packets in a tabular format, which includes:

  • Source: The device that sent the packet.
  • Destination: The device that received the packet.
  • Protocol: The communication protocol used (e.g., TCP, UDP, HTTP).
  • Length: The size of the packet.
  • Info: A brief description of the packetâ€™s content.

Step 6: Apply Filters

Wireshark allows users to apply filters to refine packet analysis. Some common filters include:

  • Filter by Protocol: To view only HTTP traffic, type http in the filter bar and press Enter.
  • Filter by Source IP: To view packets from a specific IP, use ip.src == 192.168.1.10.
  • Filter by Destination IP: To track packets sent to a specific IP, use ip.dst == 192.168.1.1.
  • Filter by TCP Flags: To view only SYN packets (connection requests), use tcp.flags.syn == 1.

Step 7: Save and Export Data

To save captured data for future analysis:

  • Click "File" > "Save As" and choose a location.
  • Select the preferred file format (e.g., `.pcap` for standard packet capture files).
  • You can analyze the saved capture offline later.

Step 8: Sample Packet Capture Output

Below is an example of a captured TCP packet: 

            Source:      192.168.1.5
            Destination: 192.168.1.1
            Protocol:    TCP
            Length:      74 bytes
            Info:        SYN Packet (Connection Request)

This information helps network administrators understand traffic patterns and detect potential security threats such as unauthorized access attempts, packet spoofing, or network anomalies.
