Theory

The purpose of this experiment is to understand how to use SQLMap to detect and exploit SQL Injection vulnerabilities in web applications.

What is SQL Injection?
SQL Injection (SQLi) is a security vulnerability that allows an attacker to manipulate SQL queries executed by a web application. It occurs when user input is improperly sanitized, enabling malicious SQL code to be injected and executed by the database.

Types of SQL Injection Attacks:

  • Union-based SQL Injection: Uses the UNION operator to extract data from other tables.
  • Error-based SQL Injection: Exploits database error messages to retrieve information.
  • Blind SQL Injection: Uses true/false conditions to extract data without error messages.
  • Time-based SQL Injection: Uses SQL functions to delay database responses, inferring data based on response time.

What is SQLMap?
SQLMap is an open-source penetration testing tool that automates the process of detecting and exploiting SQL Injection vulnerabilities. It helps security professionals assess database security and retrieve critical information.

Key Features of SQLMap

  • Automatic detection of SQL Injection vulnerabilities.
  • Database fingerprinting to determine the database type and version.
  • Data extraction, including database tables, columns, and user credentials.
  • Privilege escalation to exploit database permissions.
  • Gaining shell access through database takeover techniques.

Common SQLMap Commands and Explanations

  • Check if a website is vulnerable: 
    sqlmap -u "http://example.com/product.php?id=1" --dbs

    This command tests if the website parameter id=1 is vulnerable to SQL Injection. If it is, SQLMap lists available databases.

  • List available databases: 
    sqlmap -u "http://example.com/product.php?id=1" --dbs

    The --dbs flag tells SQLMap to retrieve all database names from the server.

  • List tables in a specific database: 
    sqlmap -u "http://example.com/product.php?id=1" -D database_name --tables

    The -D option specifies the target database, and --tables lists all tables inside it.

  • Retrieve columns from a table: 
    sqlmap -u "http://example.com/product.php?id=1" -D database_name -T users --columns

    The -T users flag specifies the users table, and --columns lists the available columns in it.

  • Extract usernames and passwords: 
    sqlmap -u "http://example.com/product.php?id=1" -D database_name -T users -C username,password --dump

    This command extracts and displays the username and password columns from the users table.

How SQL Injection Works
SQL Injection works by modifying SQL queries sent to the database:

  • The attacker injects SQL code into input fields, URLs, or headers.
  • The database executes the injected query if input validation is weak.
  • Depending on the attack type, data is retrieved, modified, or deleted.
  • If exploited, the attacker may gain full control over the database.

Preventing SQL Injection Attacks

  • Use Prepared Statements: Bind user inputs instead of concatenating them in SQL queries.
  • Sanitize Inputs: Validate and filter user input to prevent malicious injections.
  • Implement Web Application Firewalls: Block SQL injection attempts using WAFs.
  • Use Least Privilege Principle: Restrict database access rights to necessary actions only.
  • Monitor Database Activity: Detect suspicious queries using logging and anomaly detection.

By properly securing web applications, developers can prevent SQL Injection attacks and protect sensitive database information.
