Theory

The purpose of this experiment is to understand how to use IPTables to configure and manage a software firewall on a Linux host.

What is a Firewall?
A firewall is a security system that monitors and controls incoming and outgoing network traffic based on predefined rules. It acts as a barrier between a trusted internal network and untrusted external networks, such as the internet. Firewalls help in preventing unauthorized access, blocking malicious traffic, and securing sensitive data.

Types of Firewalls:

  • Packet Filtering Firewall: Examines packets and allows or blocks them based on rules.
  • Stateful Inspection Firewall: Tracks the state of active connections to determine which packets should be allowed.
  • Proxy Firewall: Acts as an intermediary between users and the internet, filtering requests.
  • Next-Generation Firewall (NGFW): Includes advanced security features like intrusion prevention and deep packet inspection.

What is IPTables?
IPTables is a Linux firewall tool that allows users to define rules for filtering, redirecting, and blocking network traffic. It operates within the netfilter framework in the Linux kernel and provides granular control over packet filtering.

Key Components of IPTables

  • Tables: Organize rules into different categories.
  • Chains: Define sequences of rules for packet processing.
  • Rules: Specify actions to be taken on packets based on conditions.

Tables in IPTables
IPTables consists of multiple tables, each serving a specific purpose:

  • Filter Table: The default table used for packet filtering (allowing or blocking traffic).
  • NAT Table: Handles Network Address Translation (NAT) for modifying source/destination addresses.
  • Mangle Table: Used for modifying packet headers (e.g., QoS settings).

Chains in the Filter Table

  • INPUT: Manages incoming packets destined for the local machine.
  • FORWARD: Controls packets routed through the system.
  • OUTPUT: Handles packets originating from the local machine.

Common IPTables Rules

  • Allow SSH traffic: iptables -A INPUT -p tcp --dport 22 -j ACCEPT
  • Block all incoming traffic: iptables -P INPUT DROP
  • Allow web traffic (HTTP & HTTPS): iptables -A INPUT -p tcp --dport 80 -j ACCEPT
  • Allow DNS resolution: iptables -A INPUT -p udp --dport 53 -j ACCEPT
  • Flush all rules: iptables -F

How IPTables Works
IPTables processes network packets based on predefined rules. When a packet arrives:

  • It is matched against the rules in the applicable chain.
  • If a rule matches, the corresponding action (ACCEPT, DROP, REJECT) is applied.
  • If no rule matches, the packet is handled based on the default policy.

Testing Firewall Rules

  • View existing rules: iptables -L
  • Check connectivity: ping -c 4 google.com
  • Test SSH access: ssh user@remote-host
  • Scan open ports: nmap -sS localhost

By properly configuring IPTables, we can create a secure environment, control access to services, and mitigate network-based attacks.
