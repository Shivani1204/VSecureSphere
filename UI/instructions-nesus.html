Nessus Vulnerability Scanning Instructions

Step 1: Login to Both Systems

Use the following credentials to log in:

  • Attacker System: Username: user, Password: VSecureSphere
  • Victim System: Username: user, Password: VSecureSphere

Step 2: Verify Network Connectivity (Attacker System)

Check the IP addresses of both systems on the attacker machine:

ifconfig

Ping the victim system from the attacker system to verify connectivity:

ping <victim_IP>

Ensure that the victim system responds to pings.

Step 3: Start and Access Nessus on the Attacker System

Start the Nessus service on the attacker system:

sudo systemctl start nessusd

Check if Nessus is running:

sudo systemctl status nessusd

Once Nessus is running, open a web browser on the attacker system and type:

http://localhost:8834

If accessing Nessus remotely, replace localhost with the attacker's IP address:

http://<attacker_IP>:8834

Ensure port 8834 is open in the firewall:

sudo ufw allow 8834/tcp

Follow these steps in the Nessus web interface on the attacker system:

  1. Go to http://localhost:8834 (or http://<attacker_IP>:8834).
  2. Click on Get Started.
  3. Choose Nessus Essentials and register with an email.
  4. Enter the activation key received via email.
  5. Wait for Nessus to install necessary components.
  6. Once installation is complete, log in with your credentials.

Step 4: Perform a Nessus Vulnerability Scan (Attacker System)

In Nessus, create a new scan:

  1. Click on New Scan.
  2. Select Basic Network Scan.
  3. Enter scan details:
    • Name: "Victim System Scan"
    • Target: <victim_IP>
    • Scan Type: Basic Network Scan
  4. Click Save & Launch to start scanning.

Step 5: Advanced Scanning Techniques (Attacker System)

Nessus allows various types of scans, including:

  • Vulnerability Mapping: Identifies weaknesses in the victim system.
  • Advanced Network Scanning: Detects misconfigurations on the victim system.
  • Port Scanning: Identifies open and vulnerable ports on the victim system.
  • Plugin-Based Scanning: Uses Nessus plugins to detect specific vulnerabilities.

Step 6: Monitor and Analyze Scan Results (Attacker System)

Once the scan is complete, check the scan report in Nessus.

Vulnerabilities are categorized as:

  • Critical (Red): Immediate threats requiring urgent attention.
  • High (Orange): Major security risks that need to be fixed soon.
  • Medium (Yellow): Moderate vulnerabilities that should be addressed.
  • Low (Blue): Minor vulnerabilities that may not be urgent.

Click on each vulnerability to view details such as affected services and recommended security fixes.

Step 7: Secure the Victim System (Victim System)

To fix vulnerabilities, connect to the victim system via SSH from the attacker system:

ssh -p 22 user@<victim_IP>

Apply security patches on the victim system:

sudo apt update && sudo apt upgrade -y

Disable unnecessary services on the victim system:

sudo systemctl stop <service_name>

Change default passwords on the victim system:

passwd

After applying fixes, re-run the Nessus scan on the attacker system to verify if vulnerabilities have been mitigated.
